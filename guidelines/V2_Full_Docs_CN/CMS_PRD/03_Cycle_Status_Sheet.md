# 1. 核心操作模块：Status Sheet (状态清单)
Status Sheet 是系统中最频繁使用的核心组件，负责集中呈现特定分发周期（Cycle）内所有的内容及其元数据配置状态。

## 1.1 核心界面与布局
Status Sheet (`HomeView.tsx`) 被设计为一个高密度的信息控制台。
- **左侧全局树 (Global Tree)**：允许运维人员按树状结构快速筛选定位栏目分类。
- **顶部过滤栏 (Filter Bar)**：提供针对航空公司的航线 (Airline)、周期 (Cycle) 等大维度的过滤。
- **动态显隐机制**：为保持业务专注度，当用户跳出内容管理进入纯系统配置页时，全局树与过滤栏将自动隐藏。

## 1.2 拖拽重排交互 (Drag & Drop)
针对松下机载屏幕严格的排序呈现要求，我们在 Status Sheet 中落地了原生的 HTML5 拖拽重排功能：
- **触发机制**：用户可长按列表行前面的拖拽手柄进行上下位移。
- **状态激活**：一旦顺序发生改变，界面右上角的蓝框操作区（如 `SAVE`, `RESET` 按钮）将由置灰状态自动激活。
- **数据持久化**：用户点击 `SAVE` 后，底层将更新 `category_media` 关联表中的 `order_position` 字段，确保下次进入或导出包时严格遵循设定顺序。
