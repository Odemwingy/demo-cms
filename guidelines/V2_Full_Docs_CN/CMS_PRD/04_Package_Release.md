# 1. 发布包导出与发布架构

包发布 (Package Release) 模块在任何数据离开 CMS 之前，负责处理所有的打包逻辑。它的主要功能是将分散的数据库记录和散落的媒体文件转化为一个单一的、标准化的发布包，完全兼容飞机上的目标 IFE 终端进行解析读取。

## 1.1 当前打包实现
目前系统主要基于轻量级的 SQLite 存储。系统的“发布包导出”现阶段主要通过后端 Node.js 脚本将 `categories` 和 `media` 结构打包输出为标准化的 JSON 描述文件，这些文件可直接用于模拟器或被装载程序读取。

# 2. DRM 打包机制 (独立模块对接)

为了支撑真实的航司下发需求，高级的加密打包逻辑被剥离为独立的开发模块，以保证系统的解耦。

## 2.1 离线 DRM 加密
高价值 AVOD 资产（例如院线大片）必须通过专用的 DRM 流水线进行处理。
- 加密引擎负责加密媒体块，并配发具备严格“时间授权窗口”（License Window）的许可证秘钥文件。
- CMS 主系统通过在 `media_configs` 和 `profiles` 中配置的安全策略结构，向 DRM 系统输出指令参数。
- 最终数字签名与加密包一同封入分发包派往机队。这套复杂的打包流不在 CMS 主工程内实现，但通过标准的接口控制文档 (ICD) 与 CMS 的导出总线集成对接。
