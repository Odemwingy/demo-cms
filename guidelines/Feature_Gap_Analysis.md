# 香港航空 IFE CMS MVP 功能落地差异与研发规划报告

基于最新修正后的《香港航空国产IFE_CMS功能清单与报价.xlsx》与《CMS 功能表 无配置功能.xlsx》功能清单，结合当前项目 (Node.js + SQLite + React 全栈 MVP) 的实际代码开发情况，逐一核对并汇总如下：

## 一、 已经开发并落地的核心功能 (已完成)
当前项目已完成核心元数据的流转与 UI 交互验证，以下模块已上线并具备演示能力：

### 1. 内容中心 (Content Center)
- **内容列表展示与检索**：✅ 已完成。通过原生弹窗完成了多列全局极速检索。
- **文件与元数据编辑**：✅ 已完成。支持查看并修改核心业务元数据（UID、标题、时长等）。
- **分类树浏览与拖拽编排**：✅ 已完成。实现了左侧无限深度的分类树，并通过原生 Drag & Drop 实现了层级挂载和排序持久化。
- **状态清单 (Status Sheet)**：✅ 已完成。作为核心工作台落地，提供清晰的业务状态鸟瞰。
- **基础多维筛选**：✅ 已完成。支持基于航线 (Airline) 和发布周期 (Cycle) 的全局过滤。

---

## 二、 后续确定需要原生开发的功能 (待开发)
根据业务规划，清单中除外部对接模块外的其余功能均需要我们在后续的迭代中进行原生开发补齐：

### 1. 用户、权限与安全架构 (IAM & Security)
- **组织、用户与角色管理**：需要开发完整的增删改查系统，替代当前的 `AuthContext` 纯前端隔离。
- **单点登录集成 (SSO)**：需要完成与航司身份中心的对接验证。
- **安全与细粒度权限**：需要研发登录日志、操作审计跟踪、只读角色，并深入到后端实现 Data Scoping (数据域行级隔离) 与接口强鉴权。

### 2. 严格的发布周期与工作流管理 (Workflow & Release Cycle)
- **协同工作台与待办中心**：需要研发支持异常提醒、里程碑节点查看与任务分发的协同首页。
- **签核工作流与 CR 变更单**：需要开发符合航空安全标准的初始技术签核、全量业务签核及锁版后的 CR 审批流机制。
- **自动化包校验与发布溯源**：需要开发在点击导出包前触发的自动化合规扫描，以及防重复导出、历史归档与包版本回滚比对功能。

### 3. 高级导入与批量操作 (Batch & Templates)
- **Excel 批量导入与预检解析引擎**。
- **分类与内容的批量变更及编排模板应用**。

---

## 三、 不进行原生开发，需通过接口外部对接的功能 (独立子系统对接)
以下涉及到重型文件物理处理的分发模块，CMS 不重复造轮子，将作为调度中枢通过接口进行对接：

### 媒体处理引擎与物理分发 (外部媒资平台 - MAM 对接)
- **物理素材上传、技术转码、DRM 加密封包与物理网络下发**：❌ **不进行原生开发**。
- **对接策略**：CMS 系统将对外提供标准化接口。当媒资系统（MAM）完成文件的物理处理与加密后，**通过唯一的文件 ID (File ID) 和唯一文件名**将处理结果与 CMS 内的元数据 (Metadata) 进行硬关联映射。

*(注：原清单中关于“外部数据同步（航司航班、机队信息等）”与“重型报表分析与大屏监控告警”等需求已被确认为无需实现且从范围中整体剔除。)*
