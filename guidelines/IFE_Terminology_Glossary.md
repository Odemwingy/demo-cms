# 航空 IFE 内容管理平台（CMS）中英文专业术语表

在航空机载娱乐（IFE）内容运营领域，由于系统复杂度高、涉及跨国硬件厂商（如 Panasonic、Thales 等）对接，大量专业名词如果直译很容易造成理解歧义。
这份文档整理了本 CMS 系统以及 IFE 行业最核心的高频专业术语标准翻译，供产品、研发和业务团队在沟通、编写文档及变量命名时作为统一基准。

## 一、 核心发布与配置流转 (Core Release & Configuration)

| 英文专业术语 | 推荐中文翻译 | 业务含义解释 |
| :--- | :--- | :--- |
| **Sign-off** | 签核 / 签核锁版 | 流程术语。指内容的最终审批确认。一旦 Sign-off，内容属性即被锁定，必须通过变更单(CR)才能修改。通常分为 Technical Sign-off（技术签核）和 Business Sign-off（业务签核）。 |
| **Profile** | 终端档案 / 终端能力基线 | 硬件与策略术语。代表飞机的软硬件能力快照（操作系统、解码能力、存储上限）。CMS 的所有发布都基于匹配相应的 Profile。 |
| **Cycle (Release Cycle)** | 发布周期 / 投放批次 | 运营术语。指一次完整的机上内容更新批次（例如 2026年6月全机队更新，或圣诞节特供）。 |
| **Manifest** | 配置清单 / 发布总清单 | 对接术语。是发往飞机的核心配置文件。里面包含了所有的媒体目录、哈希校验值和播放规则。飞机系统根据 Manifest 来装载内容。 |
| **Category (Category Set)** | 栏目分类 / 栏目结构 | 界面术语。决定了旅客在座位屏幕（Seatback Screen）上看到的菜单导航树。 |
| **Baseline** | 基线 | 流程术语。指在上一个稳定版本的基础上固化下来的配置状态，新建 Cycle 通常通过复制上一期的 Baseline 开始。 |

## 二、 媒体与版权控制 (Media & Rights Management)

| 英文专业术语 | 推荐中文翻译 | 业务含义解释 |
| :--- | :--- | :--- |
| **Metadata** | 元数据 | 内容术语。描述电影/音乐信息的文字数据（如导演、演员、简介），区别于实际的媒体文件（Media File/Asset）。 |
| **UID (Unique Identifier)** | 业务唯一标识 | 平台术语。在 CMS 中代表一个逻辑“内容资产”的永恒 ID。 |
| **MID (Media Identifier)** | 媒体唯一标识 | 平台术语。在 CMS 中代表实际生成的、附带具体分辨率/码率的“物理文件制品”。 |
| **Geo-blocking** | 地域屏蔽 / 航线限制 | 版权术语。基于飞行起降地（航线）动态隐藏没有版权的媒体内容。 |
| **DRM (Digital Rights Management)** | 数字版权保护 | 安全术语。对高清电影文件进行的加密封装，必须拥有密钥才能播放。 |
| **PPV (Pay Per View)** | 单片付费 | 运营术语。指旅客需要刷信用卡付费后才能观看的增值内容。 |
| **CSP (Content Service Provider)** | 内容服务提供商 | 合作方术语。专门为航司采购、剪辑并提供影视版权的第三方公司。 |

## 三、 流程与审核状态 (Workflow & Status)

| 英文专业术语 | 推荐中文翻译 | 业务含义解释 |
| :--- | :--- | :--- |
| **Draft** | 草稿 | 尚未提交任何审核的内容状态。 |
| **Pending / Under Review** | 待审核 / 审核中 | 内容已提交，等待各级人员校验。 |
| **Reject / Return** | 驳回 | 审核不通过，退回要求内容编辑重新修改。 |
| **Technical QC (Quality Control)** | 技术质检 | 针对音视频文件进行的机器或人工检查（码率、画质、杂音等）。 |
| **Lock / Freeze** | 锁版 / 冻结 | Cycle 或内容完成签核后的只读状态，为防止临近发布时发生数据错乱的保护机制。 |
| **CR (Change Request)** | 变更单 | 锁版后，如遇到版权纠纷或政治敏感紧急情况，用来申请特批修改的流程凭证。 |
| **Rollback** | 版本回滚 | 发布后如果导致飞机系统卡死等严重问题，紧急将飞机上的配置退回上一个稳定版本的操作。 |

## 四、 航空软硬件终端名词 (Aviation Hardware & Environment)

| 英文专业术语 | 推荐中文翻译 | 业务含义解释 |
| :--- | :--- | :--- |
| **Ground Loader** | 地面装载系统 / 装载网关 | 连接内网和飞机的硬件节点，通常是在机场机坪上的服务器或维护人员手持的 PAD，用于把 CMS 导出的包灌进飞机。 |
| **LRU (Line Replaceable Unit)** | 航线可替换单元 | 飞机硬件术语。在客舱里泛指随时可以被地勤拆卸更换的硬件模块（比如坏掉的椅背屏幕或小服务器）。 |
| **Tail Number / Registration** | 机尾号 / 注册号 | 每架飞机的唯一“车牌号”（例如 B-1823）。很多分发指令会直接下发到具体的机尾号。 |
| **Cabin Class / Class** | 客舱 / 舱位 | 指头等舱(First)、商务舱(Business)、经济舱(Economy)等，CMS 需要根据 Class 决定内容呈现。 |
| **Seatback Screen / Monitor** | 椅背屏幕 / 终端显示器 | 旅客正前方的显示器，负责最终接收和呈现 CMS 编排的 UI 菜单。 |
| **PAA (Passenger Address Announcement)** | 客舱广播 | 机组人员拿着话筒进行的全局语音播报，通常 IFE 播放器在收到 PAA 信号时必须强制静音或暂停。 |
