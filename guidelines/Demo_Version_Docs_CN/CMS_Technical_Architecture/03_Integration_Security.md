# [Airline Name] IFE CMS - 媒体集成与安全合规架构

## 1. 异构系统与外部全局集成架构
CMS 作为航空媒体内容的大脑与地面中央控制枢纽，具备生态集成能力，在保障运行隔离性与网络安全性的先决条件下，与上下游异构系统顺畅无缝地完成数据交换。

### 1.1 当前全栈集成模式
- **前后端分离与跨域集成**：前端通过 Vite 配置的代理服务器 (`localhost:2881`) 直接与后端的 Express RESTful API (`localhost:3001`) 进行跨域集成。
- **配置与资源的增量基线同步**：机载 IFE 环境单向同步控制。CMS 侧重于编排抽象逻辑结构（包括配置层级的 Category 树、内容的组合关系）。目前系统通过直接加载 SQLite `.db` 文件的方式模拟机载包下发。

### 1.2 接口报文与通信规范标准化
已通过 `AuthContext.tsx` 建立了一套前置的身份和权限验证机制，支持根据 `role`（IFE_PROVIDER, AIRLINE, CSP）在页面端进行动态路由与数据隔离。

## 2. DRM 加密与数字版权管控矩阵 (独立模块)
鉴于 [Airline Name] 需定期引进全球各大顶级院线与制片厂的高价值热映影片，好莱坞制片商及顶级版权方往往对其知识资产有着极尽苛刻的安全合规诉求。

### 2.1 离线授权打包与生命周期管理
- 航空业务特殊性在于终端往往处于完全物理断网环境。针对这一痛点，离线数字权利管理（Offline DRM Packaging）技术将作为一个完全独立的模块进行专门开发与对接。
- 加密的媒体块与数字许可证被一同封入分发包派往机队。目前我们在数据库层设计了 `media_configs` 等安全结构为 DRM 打包模块的独立接入做逻辑铺垫。

## 3. 全局信息安全体系与航司合规架构
### 3.1 当前鉴权模型
- 采用了基于 `AuthContext` 的本地角色会话验证，实现了基础的数据流保护。

### 3.2 航空数据结构合规
- 所有的架构设计（如 UID/MID 的结合，数据作用域 Scoping）均以完全符合航空飞行运行数据的安全审查为目标进行构建。
