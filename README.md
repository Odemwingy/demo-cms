# Envee CMS (Panasonic MMA Frontend/Backend Demo)

## 1. 项目概览 (Project Overview)
本项目是基于 Panasonic (松下) MMA (Media Management Architecture) 规范与数据结构重构的内容管理系统（Content Management Platform，简称 Envee CMS）。该平台实现了对航空娱乐系统 (IFE) 的音视频媒体元数据进行深度管理，并支持基于多层级航线、舱位 (Class)、设备配置 (Media Config) 等复杂拓扑结构的分发与设置。

## 2. 核心技术栈 (Technology Stack)
- **Frontend**: React 18 + TypeScript + Vite + Tailwind CSS (原生 CSS 并结合 Tailwind 布局)
- **Icons**: Lucide React
- **Backend**: Node.js + Express (提供 RESTful API)
- **Database**: SQLite3 (`server/mma.db`)

## 4. 核心数据库架构 (Database Schema Highlight)
当前的 SQLite 数据库（基于 `server/schema.sql`）针对松下 MMA 实现了如下复杂拓扑：

*   **Users & Roles (用户鉴权体系)**: `users`, `roles`, `user_roles`。
*   **Media & Metadata (媒体核心库)**: `media_items` 存储基础元数据 (Title, Duration, Status等)。
*   **Categories (分类树)**: `categories` 存储树形多层级节点数据（支持 Virtual、CID 等高级属性），与 Profile 相互映射。
*   **Profiles 体系 (结构树)**:
    1.  `profiles`: 顶层结构。
    2.  `sub_profiles` / `routes`: 子航线结构，用于多对多绑定到 Profiles。
    3.  `media_configs`: 设备映射集合，属于 Profile 的子节点。
    4.  `classes`: 舱位集合，属于 Media Config 的子节点。

## 6. 本地运行与依赖安装 (Installation & Running)

### 6.1 前提要求 (Prerequisites)
- [Node.js](https://nodejs.org/en/) (推荐 v18+ 版本)
- npm (随 Node.js 一同安装)

### 6.2 安装项目依赖 (Install Dependencies)
在项目根目录下执行以下命令以安装所有依赖（包含前端 React 相关依赖和后端 Express/SQLite 相关依赖）：
```bash
npm install
```

### 6.3 数据库初始化 (Database Initialization)
在第一次运行前，我们需要生成 `mma.db` SQLite 数据库并注入基础数据。请运行：
```bash
npm run seed
# 或者直接运行 node server/seed.js
```

### 6.4 启动项目 (Run the Project)
本项目支持在一个终端下同时启动前端页面环境和后端 API 服务（通过 `concurrently` 等库，或者分开启动）。

**分离启动方式 (Recommended for debugging):**
1. 启动后端 (Backend Server):
   ```bash
   node server/index.js
   ```
   后端服务将在 `http://localhost:3001` 上运行。

2. 启动前端 (Frontend Dev Server):
   ```bash
   npm run dev
   ```
   前端服务默认将在 `http://localhost:2881` (或者其他 Vite 默认端口) 上运行，并自动代理后端接口。

此时在浏览器中打开 `http://localhost:2881` 即可体验系统。
